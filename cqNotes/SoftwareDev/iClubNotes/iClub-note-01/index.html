Persistence

JPA OneToOne

post and postContent is a one to one relationship: a post can only have a maximum of one post content (active post content, versioned content will be in a different scenario).

See Spring Persistence Best Practices Ch1 item 11 for some tips.

It seems that if we use the regular way to map the boot to author, it creates two SQL queries that will select both book and author tables. So use the @MapsId annotation instead.

The [@MapsId] is a JPA 2.0 annotation that can be applied to [@ManyToOne] and unidirectional (or bidirectional) [@OneToOne] associations. Via this annotation, the book table’s primary key can also be a foreign key referencing the author’s table primary key. The author and book tables share primary keys (the child shares the primary key with the parent table)

JPA ManyToMany

See Spring Persistence Best Practices Ch1 item 4 (Devon link) for using @ManyToMany annotation.

For using composite primary key method (with @EmbeddedId) see Ch7 item 74 and the following item for some instructions. Also Item 76 of the same chapter describes a scenario for my use case: use a junction table for the composite primary key and with additional attributes.

Design Notes

Data

DAO defines a set of interfaces for CRUD data (which can be implemented using Spring Data JPA or JDBC, or …), then the service layer use these DAO interfaces to access or modify data, being unaware of the actual implementations.

DTO folder contains all the java class converting data entity object to domain specific objects. So the service layer (or the layer above the data access layer) is only aware of the domain specific objects and have no dependence of what the data entity looks like, thus achieving louse coupling between data access layer and the other layers. The DTO are only visible to the data layer. In addition, the data layer is fully responsible for data CRUD logic. The business layer can will translate their logic to the data layer logic.

Set all of the data fields of objects in the data folder to protected or private since they are only used within the data package.
